AI Agent 的潜力是巨大的，但我感觉现在很多人都太急了：急着往前冲，却还没看清脚下的路。

<ArticleInsight title="真正稀缺的不是会造 Agent，而是会用好 Agent。">
  <p>Agent 的价值不只在单次产出，而在于它能否被放进长期、稳定、可复用的工作流里。</p>
</ArticleInsight>

Agent 仍然处在高速发展期。光北京就有大批 AI 初创公司，最后能活下来几个？Agent 的发展一定会踩在大量失败项目的经验之上，就像互联网发展初期一样，大浪淘沙，能留下来的永远是少数。

而且，从“使用 Agent”到“用好 Agent”，中间还有很长一段距离；从“开发 Agent”到“Agent 效果好”，更是一道鸿沟。这两步之间的差距，很多人低估了。

拿 Coding Agent 来举例。目前无论是 Codex、Claude Code，还是国内的 Trae、CodeBuddy，不同工具的体验有差异，但底层逻辑大体相似。这不影响下面的判断。

作为一名互联网开发从业者，我比较了解代码开发这个行当。代码这个行当有个特点：**同质化的编码能力从来就不怎么值钱，真正值钱的是业务理解。** 这个逻辑放到今天依然没变。

AI 真正冲击的，是那些模板化的、不需要深度判断的编码劳动。这部分能力的价值正在快速下降。但业务理解、架构判断、技术选型、长期维护，这些事情依然要人来兜底。

现在的 Coding Agent 开发一个小工具完全没问题，甚至可以说手到擒来。但如果要开发一个可持续迭代的项目，就不是“把需求丢给 Agent”这么简单了。问题出在哪？我观察到至少两点。

## 第一，结构失控

Coding Agent 在没有人类架构约束的前提下，天然倾向于产出“能跑就行”的代码。第一次看起来很快，第二次还能凑合，迭代几轮之后，就没人想碰了。

我觉得写代码跟写书很像。如果没有结构认知来约束，写出来的代码就像一份不断叠加的草稿。这本书能看吗？能看。但你想看吗？翻开之后，一片乱糟糟：第一章写着本该在第二章展开的内容，第二章又重复第一章已经说过的话。整本书毫无阅读逻辑，日后还怎么续写下去？

整洁的逻辑不仅是为了让读者能读下去，更是为了让写作者自己能持续往下写。代码也是一样。好的结构不是洁癖，而是让项目能继续生长的基础。

## 第二，重复造轮子

Coding Agent 如果不知道项目里已经有什么，就经常凭空再造一套。一次两次看不出问题，时间久了，代码库就会越来越臃肿。

打个比方，本来“风和日丽”四个字就能说清的事，它非要写成“今天处于一个天气状况良好、日照充足、云量稀少、气温适中、微风徐徐、体感舒适、能见度极佳、适宜户外活动的气象条件日”。

映射到代码上就是：明明项目里已有模块可以复用，或者成熟的开源方案就摆在那里，它却非要自己手写一套，而且还不一定比现成方案好用。

有人会说，这很好解决啊，在提示词里加约束不就行了吗？但现实是，提示词能承载的上下文是有限的。你不可能把整个项目的架构设计、已有模块清单、命名规范、依赖关系，全部塞进一段说明里。

更何况，Agent 对长篇约束的遵循程度本身就不稳定。前面几条还记得，写着写着就偏了。说到底，这就是“使用 Agent”和“用好 Agent”之间那道鸿沟的具体表现。

其实不止 Coding Agent 有这个问题。任何 Agent 只要面对的是“可持续产出的复杂任务”，都会踩到类似的坑。写作 Agent 可能会写出逻辑混乱的长文，数据分析 Agent 也可能在已知结论上绕圈子。Coding Agent 只是把问题暴露得最干净而已，因为代码质量是客观可衡量的：编译不过就是过不了，结构烂就是结构烂，骗不了人。

> 冲刺的前提，是知道终点在哪、路上有什么坑。

话说回来，我并不是在给 Agent 泼冷水。恰恰相反，正因为我相信它的潜力，才觉得现在应该慢一点。如果连脚下的路都没看清就跑，摔的是自己，耽误的可能是整个行业。

互联网泡沫崩掉之后，活下来的那批公司才真正改变了世界。Agent 也是一样。死掉的大部分会被遗忘，但活下来的那几个，会把“用好 Agent”这件事做成基础设施。

**所以接下来我想做的事，不是去搭一个什么 Agent。**

对我来说，造一个 Agent 是“搭房子”的工程。那需要大量的资源、团队和底层能力，一个人很难把那块砖搬起来。

我真正想做的，是成为那个“把 Agent 用得很好”的人。

把 Agent 用好，远不止是写好提示词那么简单。提示词是基本功，但更重要的是另一层判断力：什么样的任务拆成 Workflow 就能解决？什么样的 Workflow 结构最稳？什么样的场景一个 Skill 就能收口？怎么在已有 Agent 上做扩展、做衔接？

这些问题没有银弹。答案只能来自大量实践：一条一条总结经验，把踩过的坑、验证过的方法、稳定可复用的流程沉淀下来。这才是我说的“用好”。

工具已经摆在那了。

能不能用好，看人。

别急着冲，先把脚下的路看清。Agent 是好东西，但路还长。
